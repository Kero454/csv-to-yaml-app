Step 4: Architecture Files - Help
What are Architecture Files?

Architecture files define the structure and configuration of your machine learning models. They specify model parameters, layers, training settings, and other crucial aspects of your ML pipeline.

Available Options:
Option 1: Upload

Upload existing architecture files (.yaml, .yml, .json) that you already have configured.

Best for: Users with pre-existing architecture definitions

Option 2: Configure

Use our guided interface to create architecture files step-by-step with validation and smart defaults.

Best for: Users who want guided architecture creation

Option 3: YAML Training

Advanced YAML configuration for training models with comprehensive parameter settings.

Best for: Advanced users needing detailed training configurations

Architecture File Contents:
  • Model Type: Specify the ML model (LSTM, Transformer, etc.)
  • Layer Configuration: Define network architecture
  • Hyperparameters: Set learning rates, batch sizes, etc.
  • Training Settings: Configure epochs, validation, etc.
  • Data Processing: Define input/output specifications
Pro Tip: If you're new to ML architecture configuration, start with Option 2 (Configure) for guided assistance. Experienced users can upload existing files with Option 1.
